# Laba2

## Установка и запуск

### 1. Создание файла окружения
Создайте файл `.env` на основе шаблона:

```bash
cp .env.example .env
```

После этого отредактируйте файл `.env`, добавив необходимые значения (например, пароли, ключи API и т.д.).

### 2. Сборка и запуск проекта
Соберите и запустите контейнеры с помощью Docker Compose:

```bash
docker compose up
```

Эта команда запустит все сервисы в фоновом режиме (используйте `docker compose up -d` для демонизации). Для остановки используйте `docker compose down`.

### 3. Восстановление дампа базы данных
Если у вас есть файл дампа `backup.sql`, восстановите базу данных:

```bash
cat backup.sql | docker exec -i laba2-db-1 psql -U postgres
```

Убедитесь, что контейнер `laba2-db-1` запущен перед выполнением команды. Если база данных пустая, это заполнит её данными из дампа.

## Дополнительная информация
- **Зависимости**: Убедитесь, что Docker и Docker Compose установлены.
- **Порты**: По умолчанию проект доступен на `http://localhost:8000` (адаптируйте под ваш `.env`).
- **Логи**: Просмотр логов: `docker compose logs -f`.

Если возникнут проблемы, проверьте логи контейнеров или обратитесь к разделу [Troubleshooting](troubleshooting.md).
